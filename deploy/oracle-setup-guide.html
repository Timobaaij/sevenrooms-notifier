Part 0

Before you start

Have these three things ready — you'll paste them in near the end:

  1. The Gmail address that sends your alerts

    The same one you use in GitHub today (e.g. you@gmail.com).

  2. A Gmail App Password (16 characters)

    Not your normal password. If you don't have one yet, don't worry — Part 5 walks you through creating it.

  3. Your Pushover email-to-push address (optional)

    The one your notifier already emails for push alerts. Leave blank if you only want email alerts.

Good to know

You never touch code. You'll create a tiny Linux computer in Oracle's cloud, paste one install command, fill in a small settings file, and you're done.

Part 1

Create an Oracle Cloud account

  1. Go to the free-tier sign-up page

    Open oracle.com/cloud/free and click Start for free.

  2. Enter your country and email

    Pick your real country, type your email, and click Verify my email. Open the email from Oracle and click the verification link.

  3. Set your password and account name

    Create a password. The Cloud Account Name is a unique label for your tenancy (e.g. timo-cloud) — remember it, you'll need it to sign in.

  4. Choose your home region — carefully
    Can't be changed later

    Pick the region closest to you (e.g. UK South (London), Germany Central (Frankfurt), or Amsterdam). Your home region is permanent, and Always Free capacity varies by region.

  5. Verify by phone

    Enter your mobile number and type the SMS code.

  6. Add a payment method (identity check only)
    You will not be charged

    Oracle asks for a card to confirm you're human. Always Free resources are free forever and never billed. You may see a temporary ~£1 authorisation hold that reverses itself. As long as you stay on the free tier, the amount owed is £0.

  7. Finish and wait for provisioning

    Click Start my free trial. Setup takes a few minutes; you'll get an email when your account is ready, then land in the OCI Console at cloud.oracle.com.

Part 2

Create the free VM

You're now in the OCI Console. We'll create one small Always Free Ubuntu machine.

  1. Open the instance creation page

    Click the ≡ navigation menu (top-left) → Compute → Instances. Then click Create instance.

  2. Name it & leave placement default
    Name
    maitre-notifier (anything you like)
    Compartment
    Leave as-is (your root compartment)
    Placement / Availability domain
    Leave the default
    Capacity type (Advanced)
    Leave On-demand (the default)
  3. Switch the image to Ubuntu

    In the Image and shape section, click Change image. In the panel:

    • Choose Platform Images
    • Select Canonical Ubuntu
    • Pick version 22.04 (24.04 is fine too)
    • Tick the row and click Select image
  4. Choose the free shape

    Click Change shape. In the Browse all shapes panel:

    • Instance type: Virtual machine
    • Shape series: click Specialty and previous generation
    • Select VM.Standard.E2.1.Micro — it carries an Always Free-eligible label
    • Click Select shape
    Why this shape

    E2.1.Micro (1 GB RAM) is tiny but more than enough for a once-a-minute poll, and it's the most reliably available free shape. The Arm shape VM.Standard.A1.Flex is beefier and also free, but often returns "Out of host capacity" — try it only if you want more power (see Troubleshooting).

  5. Networking — keep the defaults, keep the public IP

    Under Networking, let it Create a new virtual cloud network. Confirm Assign a public IPv4 address is set to Yes (it is, on a public subnet). You do not need to open any ports — the watcher only makes outbound connections, and SSH (port 22) is open by default.

  6. Create an SSH key — and SAVE the private key

    In Add SSH keys, the simplest option is Generate a key pair for me. Then:

    • Click Save private key — this downloads a file like ssh-key-2026-07-05.key
    • Click Save public key too (optional backup)
    Do this now — no second chance

    You can't re-download the private key later. Keep ssh-key-2026-07-05.key somewhere safe (e.g. your Downloads or a password manager). It's the only key to your server.

    Already have your own SSH key? Choose Paste public keys instead and paste your .pub.

  7. Create it

    Leave Boot volume at its defaults. Click Create. After ~1–2 minutes the status turns Running.

  8. Copy the public IP address

    On the instance details page, find Public IPv4 address (under Instance access / Primary VNIC). Copy it — e.g. 140.238.x.x. You'll use it next.

Part 3

Connect over SSH

Now open a terminal on your own computer and log into the server. The username for Ubuntu images is always ubuntu.

  1. macOS or Linux

    Open Terminal. Lock down the key's permissions, then connect (swap in your key's filename and your server's IP):

    chmod 400 ~/Downloads/ssh-key-2026-07-05.key
ssh -i ~/Downloads/ssh-key-2026-07-05.key ubuntu@<YOUR_PUBLIC_IP>
  2. Windows

    Open PowerShell or Windows Terminal (SSH is built in) and run:

    ssh -i C:\Users\YOU\Downloads\ssh-key-2026-07-05.key ubuntu@<YOUR_PUBLIC_IP>

    If Windows complains the key is too open, run this once, then retry the ssh command:

    icacls C:\Users\YOU\Downloads\ssh-key-2026-07-05.key /inheritance:r /grant:r "%USERNAME%:R"
  3. Accept the fingerprint

    The first time, it asks "Are you sure you want to continue connecting?" — type yes and press Enter. You're in when the prompt reads ubuntu@maitre-notifier:~$.

Can't connect?

Jump to Troubleshooting → SSH. Usual causes: wrong key filename, wrong IP, or you used a username other than ubuntu.

Part 4

Install the watcher

Everything from here runs on the server (in that SSH window). Paste these three lines one after another:

git clone -b claude/iphone-app-feasibility-788w8h https://github.com/timobaaij/sevenrooms-notifier.git
cd sevenrooms-notifier
bash deploy/setup.sh

The script installs Python, its dependencies, and a background service that runs your watcher 24/7. It takes a minute or two and prints "Done." when finished.

Once this is merged to your main branch

You can drop the -b claude/iphone-app-feasibility-788w8h flag and just clone normally.

If your repository is private

A private repo needs a token to clone. Create a fine-grained token at github.com/settings/tokens → limited to this one repo → Contents: Read-only. Then clone with the token in the URL (it's saved so the hourly git pull keeps working):

git clone -b claude/iphone-app-feasibility-788w8h https://YOUR_TOKEN@github.com/timobaaij/sevenrooms-notifier.git

Everything else is identical.

Part 5

Add your secrets

  1. Open the settings file 
    nano ~/.maitre/notifier.env

    This opens a simple text editor. Fill in your values after each = sign:

    EMAIL_USER
    Your sending Gmail address
    EMAIL_PASS
    Your 16-char Gmail App Password (below)
    PUSHOVER_EMAIL
    Your Pushover push address (or leave blank)
    EMAIL_TO
    Optional fallback address for email alerts
    POLL_INTERVAL
    60 = every minute
  2. Save and exit nano

    Press Ctrl+O then Enter to save, then Ctrl+X to quit.

  3. Restart the watcher so it picks up your secrets 
    sudo systemctl restart maitre-notifier

Creating a Gmail App Password

  1. Turn on 2-Step Verification

    Go to myaccount.google.com/security → enable 2-Step Verification if it isn't already. (App Passwords require it.)

  2. Generate the App Password

    Open myaccount.google.com/apppasswords, name it maitre, and click Create.

  3. Copy the 16 characters

    Paste them (no spaces) as your EMAIL_PASS value in the settings file above.

Part 6

Confirm it's running

Watch the live log:

journalctl -u maitre-notifier -f

You should see a line roughly every 60 seconds as it polls. Press Ctrl+C to stop watching the log (this does not stop the watcher). A quick health check any time:

systemctl status maitre-notifier

Look for active (running). That's it — your watcher now runs every minute, forever, for free, and comes back on its own after any reboot.

Test it end-to-end

Add a search in your web dashboard for a time you know is available, wait a minute, and confirm the alert lands. Then set it back to what you actually want.

Part 7

Housekeeping & keeping it free

  1. Turn off GitHub Actions (avoid double-polling)

    In your repo on GitHub: Settings → Actions → General → Disable actions. (Or delete .github/workflows/sevenrooms-notifier.yml.) Your VM is the watcher now.

  2. Keep the free VM from being reclaimed
    Important for Always Free

    Oracle may reclaim an idle Always Free machine (CPU & network under 20% for 7 days). A once-a-minute poll uses very little, so to be safe do one of these:

    • Best: In the Console, upgrade the account to Pay As You Go. This keeps everything free within the Always Free limits but exempts your instance from idle-reclamation. (You're only ever charged if you exceed free limits — this tiny VM won't.)
    • Or: do nothing and simply re-run this guide if it's ever reclaimed (rare, and your searches live safely in GitHub).
  3. Updating later

    When the code changes, SSH in and run:

    cd ~/sevenrooms-notifier
git pull
bash deploy/setup.sh
sudo systemctl restart maitre-notifier
  4. Pause / resume the watcher 
    sudo systemctl stop maitre-notifier     # pause
sudo systemctl start maitre-notifier    # resume

Troubleshooting

SSH won't connect / "Permission denied" / timeout
  • Username: it must be ubuntu for Ubuntu images (not root or your name).
  • Key path & name: confirm the filename matches what you downloaded and the path is correct.
  • Permissions (Mac/Linux): run chmod 400 on the key first.
  • Right IP: re-copy the Public IPv4 address from the instance page.
  • Timeout: in the Console open your instance → its Subnet → Security List → confirm an ingress rule allows TCP 22 from 0.0.0.0/0 (present by default).
"Out of host capacity" when creating the instance

This almost always happens with the Arm A1.Flex shape. Fixes, in order:

  • Use VM.Standard.E2.1.Micro instead (Part 2, step 4) — it's rarely capacity-constrained.
  • If you specifically want A1: try a different Availability domain in the dropdown, or retry later — free Arm capacity frees up in waves.
No alerts arriving
  • Check the log for errors: journalctl -u maitre-notifier -n 50.
  • Gmail: make sure EMAIL_PASS is an App Password, not your normal password, and that you saved the file and restarted the service.
  • Re-open nano ~/.maitre/notifier.env and check for typos / stray spaces, then sudo systemctl restart maitre-notifier.
  • Confirm the search actually has availability at the requested time.
Service shows "failed" or keeps restarting

Read the last lines: journalctl -u maitre-notifier -n 80 --no-pager. If dependencies didn't install, re-run bash deploy/setup.sh. If Python errors mention a missing package, run ~/sevenrooms-notifier/.venv/bin/pip install -r ~/sevenrooms-notifier/requirements.txt then restart.

My searches on the VM don't match the web dashboard

The loop runs git pull each minute to fetch edits you make in the web app. If your repo is private and you cloned without a token, that pull can't authenticate — see Part 4 → private repository, re-clone with a token, and re-run setup.

  • Always Free VM.Standard.E2.1.Micro Ubuntu instance is Running
